### Input {#input}

Name: `pull-request-days-before-stale`  
Type: `number`  
Default value: `30`

### Description {#description}

This input will let you define the number of days before processing the pull requests as stale.

The number of days is calculated based on the last update that occurred on the pull requests.

**For example:**

Assuming that a pull request is inactive since 10 days and this input is configured to stale after two weeks.  
Someone adds a comment on this pull request.  
The pull request cannot be stale until at least 2 weeks once again.

### Example {#example}

```yml {6}
# ...
name: Stale
id: stale
uses: sonia-corporation/stale@latest
with:
  pull-request-days-before-stale: 14
```
