### Input {#input}

Name: `pull-request-add-labels-after-stale`  
Type: `string[]`  
Default value: `[]`

### Description {#description}

This option will let you add extra labels when the processing stale the pull requests.  
This can be useful if you wish to easily add extra labels to improve your triage post-stale.

:::caution

The labels must be real labels, existing inside your repository list of labels (_github.com/your-organization/your-repository/labels_).  
If not, the GitHub API will throw an error.

:::

### Example {#example}

```yml {6-8}
# ...
name: Stale
id: stale
uses: sonia-corporation/stale@latest
with:
  pull-request-add-labels-after-stale: |
    warning
    triage obsolete
```
