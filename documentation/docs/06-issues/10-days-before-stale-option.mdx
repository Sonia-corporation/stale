### Input {#input}

Name: `issue-days-before-stale`  
Type: `number`  
Default value: `30`

### Description {#description}

This option will let you define the number of days before processing the issues as stale.

The number of days is calculated based on the last update that occurred on the issues.

**For example:**

Assuming that an issue is inactive since 10 days and this option is configured to stale after two weeks.  
Someone adds a comment on this issue.  
The issue cannot be stale until at least 2 weeks once again.

### Example {#example}

```yml {6}
# ...
name: Stale
id: stale
uses: sonia-corporation/stale@latest
with:
  issue-days-before-stale: 14
```
