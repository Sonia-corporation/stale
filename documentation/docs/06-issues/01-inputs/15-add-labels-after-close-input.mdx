### Input {#input}

Name: `issue-add-labels-after-close`  
Type: `string[]`  
Default value: `[]`

### Description {#description}

This input will let you add extra labels when the processing close the issues.  
This can be useful if you wish to easily add extra labels to improve your triage post-closing.

:::caution

The labels must be real labels, existing inside your repository list of labels (_github.com/your-organization/your-repository/labels_).  
If not, the GitHub API will throw an error.

:::

### Example {#example}

```yml {6-8}
# ...
name: Stale
id: stale
uses: sonia-corporation/stale@latest
with:
  issue-add-labels-after-close: |
    do not reopen
    please-create if needed
```
