### Input {#input}

Name: `issue-ignore-all-labels`  
Type: `boolean`  
Default value: `false`

### Description {#description}

This option will let you ignore the processing of the issues which have at least one label.  
This can be useful when you use labels for triage.

If you use some default labels in your [issues templates](https://docs.github.com/en/communities/using-templates-to-encourage-useful-issues-and-pull-requests/configuring-issue-templates-for-your-repository#creating-issue-forms), this option may not suit you.  
You may prefer to use instead the [ignore any labels option](issue-ignore-any-labels-option) to list only some specific labels.

:::note

If the issue is stale and so, has a [stale label](issue-stale-label-option), this option will not ignore the processing.

:::

### Example {#example}

```yml {6}
# ...
name: Stale
id: stale
uses: sonia-corporation/stale@latest
with:
  issue-ignore-all-labels: true
```
